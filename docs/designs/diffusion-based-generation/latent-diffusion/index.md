# Latent Diffusion World Models — 隐空间扩散世界模型

::: tip 定义
Latent Diffusion World Models 将扩散去噪过程运行在**压缩的隐空间**而非原始像素空间。先用 VAE 将高维视觉观测编码为紧凑的隐表示，在隐空间中进行条件和去噪，最后通过解码器恢复像素。这是当前扩散 VWM 的**主流技术路线**，在计算效率和生成质量之间取得了最佳平衡。
:::

## 为什么需要隐空间扩散？

在原始像素空间中直接运行扩散模型有两个致命问题：

1. **计算不可行**：一个 256×256×3 的视频帧有约 200k 个像素。扩散模型（通常基于 UNet 或 DiT）需要在每一步去噪中处理整个视频序列的全分辨率数据。假设 16 帧、50 步去噪——那就是 50 × 200k × 16 ≈ 160M 次全分辨率的矩阵运算，单张 GPU 几乎不可能完成。

2. **语义效率低**：像素空间的绝大部分自由度用于表示纹理细节、光照等「表面特征」，而非世界状态的「本质信息」。直接在像素空间中做条件生成，模型会把大量容量浪费在对下游任务无用的视觉细节上。

隐空间扩散通过一个预训练（或联合训练）的 VAE 将像素压缩到低维隐空间（典型压缩比 8×-48×），从而将以上两个问题各缓解 1-2 个数量级。

## 核心演进脉络

### DriveDreamer（2023-2024）

DriveDreamer 是**自动驾驶领域首个系统性的隐空间扩散世界模型**。其核心贡献在于证明了扩散 VWM 在自动驾驶场景中的可行性和优越性：

- **多条件融合**：将地图、3D 检测框、HD 地图、自车轨迹等多种条件信号统一嵌入到扩散过程中
- **多视角一致性**：通过跨视角注意力机制保证环视相机生成的一致性
- **闭环评测**：在 nuScenes 数据集上展示了闭环规划能力的初步验证

DriveDreamer 的重要发现是：**扩散 VWM 在生成交通参与者的「social behavior」（如其他车辆的变道、让行）时，表现出优于规则模型和传统预测模型的合理性**——这表明扩散模型在隐空间中确实学到了交通参与者之间的交互模式。

### GenAD（2024）

GenAD（Generative Autonomous Driving）将扩散 VWM 推进到**端到端自动驾驶**的语境中。其架构特点：

- **Unified World Model + Planner**：同一个扩散模型同时承担世界模拟和轨迹规划两个角色
- **Trajectory as Condition**：规划的轨迹嵌入作为去噪条件，生成该轨迹下的未来场景
- **Safety-Critical Evaluation**：通过扩散 VWM 发现规划轨迹可能引发的危险场景

GenAD 展示了一个重要趋势：**扩散 VWM 正在从「感知辅助」走向「规划核心」**——世界模型不再仅仅帮助感知系统理解当前场景，而是直接参与决策过程。

### Cosmos（2025, NVIDIA）

Cosmos 是当前**规模最大、最全面的隐空间扩散 VWM**。NVIDIA 将其定位为「物理 AI 的世界基础模型」。核心特点：

- **3D VAE**：使用 3D（时空）VAE 进行联合时空压缩，而非 2D VAE 逐帧编码。具有更好的时序一致性
- **多领域预训练**：在自动驾驶、机器人操作、物流导航等多个物理领域的视频数据上联合训练
- **大规模**：参数规模达到数十亿级别，训练数据覆盖数千万小时的物理世界视频
- **可控生成**：支持多种条件信号——文本描述、参考帧、轨迹、3D 场景布局

Cosmos 的 3D VAE 设计是该工作的核心技术贡献。传统的 2D VAE 逐帧压缩视频会导致隐空间中相邻帧的隐向量之间存在不连续跳跃，而 3D VAE 通过 3D 卷积同时处理时空维度，自然保证了隐空间的时序平滑性。这对于世界模型的质量至关重要——因为在隐空间中的扩散过程必须生成时序上连续的隐状态序列。

### GAIA-2（2025, Wayve）

GAIA-2 代表了**自动驾驶专用扩散 VWM 的工程化巅峰**。Wayve 将其部署在实际的端到端自动驾驶系统中：

- **LINGO-2 集成**：将扩散 VWM 与视觉语言模型（LINGO-2）深度集成，实现语言可解释的世界模拟
- **实时推理**：通过模型蒸馏和专用推理优化，实现了接近实时的世界模拟（推理延迟约 200ms）
- **安全验证**：扩散 VWM 被用于在线评估规划轨迹的安全性——模拟「如果执行这个轨迹，接下来 3 秒世界会变成什么样」

GAIA-2 的实践表明：**扩散 VWM 在工程化之后，完全可以作为自动驾驶决策的核心组件**，而非仅仅是离线的数据增强工具。但其推理延迟（200ms）仍远高于 State Transition 模型（通常 <10ms），这是其作为实时决策组件时需要警惕的。

### 机器人领域的扩散 WMs

在机器人领域，隐空间扩散 VWM 同样蓬勃发展：

- **UniPi**（Google）：提出将图像扩散模型直接作为通用策略——给定当前观测和目标图像，生成动作序列使观测趋向目标
- **Susie**：探索在机器人操作中使用扩散模型进行「视觉规划」——预测操作后的场景变化
- **Diffusion Policy** 系列：将扩散模型用于机器人动作生成，虽然不是完整的世界模型，但其「从噪声中逐步去噪得到动作序列」的思路与扩散 VWM 一脉相承

机器人扩散 WMs 面临的核心挑战是**推理速度与操作精度的冲突**：扩散模型的生成质量高但速度慢，而机器人需要毫秒级的实时控制。这催生了对蒸馏和加速方向的大量研究。

## 隐空间扩散的核心技术组件

### 1. VAE 架构选择

隐空间的质量完全取决于 VAE。当前主流 VAE 方案：

| VAE 类型 | 时空处理 | 压缩比 | 隐空间维度 | 代表工作 |
|---------|---------|--------|-----------|---------|
| **2D VAE** | 逐帧独立 | 8× | 4 channels | Stable Diffusion 系列 |
| **3D VAE** | 联合时空 | 4×8×8 | 16 channels | Cosmos, Sora |
| **Causal 3D VAE** | 因果时序 | 4×8×8 | 16 channels | MAGVIT-v2 |
| **Video VAE** | 时序注意 | 8×8×16 | 4 channels | CV-VAE |

3D VAE 是扩散 VWM 的首选——它保证了隐空间的时序连续性。但 3D VAE 的训练比 2D VAE 困难得多（需要大规模视频数据且容易过拟合），因此许多工作仍采用分阶段策略：先训练 2D VAE，再通过时序模块微调。

### 2. 噪声调度与采样策略

扩散过程中的噪声调度（noise schedule）直接决定了生成质量。对于 VWM 而言，特别需要关注的是**时序一致性相关的噪声设计**：

- **视频扩散模型通常采用 shifted noise schedule**——在高噪声阶段更激进地去噪（快速恢复全局结构），在低噪声阶段更保守地去噪（精细恢复细节）。这有助于保持视频帧间的一致性。

- **DDIM 采样**是最常用的加速采样策略，通过确定性的 ODE 求解器将 50-100 步去噪减少到 20-50 步。

- **Rectified Flow** 是更新的趋势——通过学习噪声和干净数据之间的直线路径，用更少的采样步数达到相同质量。

### 3. 条件注入架构

扩散 VWM 的条件注入有多种设计方案：

- **Cross-Attention Conditioning**：在每个去噪 transformer 层中，隐状态对条件嵌入做 cross-attention。最灵活也最昂贵。

- **Additive Conditioning**：将条件嵌入与时间步嵌入相加后注入。简单但控制力有限。

- **Adaptive Norm Conditioning**：用条件信号调制 layer norm 的参数（scale 和 shift）。在效率和灵活性之间取得平衡，Cosmos 和 GAIA-2 均采用此方案。

- **Concat Conditioning**：将条件信号与隐状态在通道维度拼接。在自动驾驶的 BEV 图、地图条件中尤为常见。

## 隐空间扩散的优势与局限

**优势**：
- 生成质量卓越（目前所有 VWM 范式中视觉质量最高的）
- 计算相对高效（隐空间压缩使得大规模模型可行）
- 强大的多条件融合能力
- 通过 CFG 可以实现精确的可控性调节
- 与视频生成技术的协同发展（可受益于 Sora 等通用视频模型的技术进步）

**局限**：
- 推理速度仍不足以支撑实时控制（>100ms）
- 物理一致性不如 State Transition 模型
- VAE 解码引入的失真可能影响精确的物理交互任务
- 训练需要海量视频数据
- 隐空间的不可解释性——难以审计世界模型「理解」了什么
