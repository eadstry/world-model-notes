## 学习笔记

## 核心思想

RoboScape（Physics-informed Embodied World Model）由清华大学提出，针对当前具身世界模型普遍缺乏物理感知的问题，提出了一个**物理信息融合的具身世界模型**。现有世界模型在模拟接触密集型机器人操作场景（如抓取、推动、插拔）时，往往生成不符合物理规律的视频——物体穿透、不合理形变、运动轨迹违反动量守恒等。RoboScape 的核心贡献是**在 RGB 视频生成任务中联合学习物理知识**，使得生成的视频不仅视觉上逼真，同时在物理层面合理。

RoboScape 引入两类物理信息联合训练任务：(1) **时序深度预测**——预测未来帧的深度图，强化 3D 几何一致性；(2) **关键点动力学学习**——预测物体表面关键点的运动轨迹，隐式编码物体的材质属性（硬度、弹性）和形状特征。这两种物理信号作为 RGB 视频生成任务的辅助监督，使模型在生成 RGB 帧时"考虑到"3D 几何约束和物理动力学规律。这种物理感知能力的提升在接触密集型操作场景中尤为显著——物体之间的交互（如夹爪与物体的接触）在物理上更加合理。

## 技术架构

**Vision Encoding（视觉编码）**：RoboScape 使用多模态编码器，同时处理 RGB 图像和深度图像。RGB 流通过 CNN/ViT 编码为视觉 token，深度流通过专门的深度编码器（与 RGB 编码器结构类似但参数独立）编码为深度 token。两者通过交叉注意力或拼接方式融合，形成富含 3D 几何信息的综合视觉表示。

**Knowledge Learning（知识学习）**：RoboScape 的训练包含三个并行的损失项：(1) **RGB 视频重建损失**——标准的像素级重建损失或感知损失；(2) **深度预测损失**——将未来帧的深度预测与真实深度（从 RGB-D 数据获取）对齐，使用 L1/L2 损失和结构相似性损失；(3) **关键点动力学损失**——在物体表面采样关键点（如角点、边缘点），预测这些关键点在未来帧中的像素坐标，使用 L2 距离损失。三个损失加权联合训练确保模型既重视视觉质量又重视物理一致性。

**Controllable Simulation（可控模拟）**：RoboScape 的动作控制与标准动作条件视频生成一致。附加的物理感知使其在需要精细物理交互的场景中表现更好：(1) 接触检测——生成的视频中夹爪和物体的接触时点在物理上合理；(2) 形变一致性——柔性物体（如布料）的形变符合物理规律；(3) 碰撞避免——生成中不产生穿透现象。

## 代码实现要点

代码已在 GitHub 开源（tsinghua-fib-lab/RoboScape）。核心实现：(1) 共享的 RGB + 深度编码器（基于 ViT）；(2) 自回归视频生成 Transformer 主干；(3) 三个并行的解码头（RGB、深度、关键点）；(4) 多任务加权训练管道。深度数据来自仿真环境中的 RGB-D 渲染或真实机器人操作中的深度相机。关键点的选取使用预训练的 SuperPoint 或手动标注。代码支持机器人策略训练（使用生成的数据增强训练集）和策略评估（在生成的世界中闭环测试策略表现）。

## 关键创新点

- **第一个系统性研究物理感知世界模型**：明确定义了世界模型中的物理感知问题，提出了系统性的解决方案
- **深度+关键点双物理监督**：同时利用 3D 几何信息（深度）和运动动力学（关键点）提升物理一致性
- **接触密集型场景突破**：在抓取、推动、插拔等需要精细物理交互的场景中实现了显著更优的视觉效果和物理合理性
- **下游应用验证**：不仅提升了视频生成质量，还在机器人策略训练和评估等下游任务中创造了实际价值

## 代表性结果

在多种接触密集型机器人操作场景（如抓取易拉罐、推动滑块、插拔插头）中，RoboScape 生成的视频在 FVD、PSNR/SSIM 等视觉质量指标上显著优于没有物理监督的基线。更重要的是，人工评估中生成视频的物理合理性（如"物体是否穿透""运动是否自然"）评分提升明显。在下游应用中，使用 RoboScape 生成数据训练的机器人策略在成功率上相比使用基线生成数据提高了约 15-20%。消融实验确认了深度预测和关键点动力学两个任务各自的贡献——两者联合使用效果最优。
