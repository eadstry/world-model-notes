## 学习笔记

### 核心思想

GWM (Graph World Model) 是一种将世界模型范式扩展到图结构数据的新框架。现有世界模型主要关注非结构化数据（图像、视频），无法利用数字世界中普遍存在但以图形式表示的丰富结构化信息。虽然已有多个图基础模型被提出，但它们聚焦于图学习任务，无法扩展到多样化的多模态数据和跨学科任务。GWM 的核心创新是：通用消息传递算法聚合结构化信息——要么在统一的多模态 token 空间中将多模态数据转换为文本（GWM-T），要么在统一的多模态嵌入空间中使用模态专用编码器（GWM-E）。更重要的贡献是引入动作节点（action nodes）——将多样化任务统一表示为图中连接到其他节点的动作节点（通过直接引用或相似度计算），使同一模型能够处理多模态生成和匹配、推荐系统、图预测、多智能体模拟、检索增强生成以及规划和优化等六类完全不同域任务。

### 技术架构

**Vision Encoding（视觉编码）**：GWM 通过两种方式处理多模态数据：(1) GWM-T——将图像、视频等通过预训练 VLM/图描述器转换为文本 tokens，所有模态统一在语言 token 空间中；(2) GWM-E——使用模态专用编码器（如 ViT for images, Graph Encoder for graphs）映射到统一的多模态嵌入空间。这种灵活性使 GWM 能处理从视觉到文本到图的任意组合输入。

**Knowledge Learning（知识学习）**：通用消息传递算法是 GWM 的计算核心。图节点间的信息通过注意力边（attention edges）传递，但与传统图神经网络不同，GWM 中的边可以是动态构建的——动作节点通过直接引用（硬连接）或相似度计算（软连接）与相关节点关联。这使得模型可以自动学习哪些节点需要交互，而不需要预设图结构。任务多样性通过动作节点实现：不同任务被定义为不同的动作节点类型，模型学习根据动作节点类型激活相应的消息传递路径。

**Controllable Simulation（可控仿真）**：GWM 在六类跨域任务中展示能力：(1) 多模态生成和匹配；(2) 推荐；(3) 图预测；(4) 多智能体模拟；(5) 检索增强生成；(6) 规划和优化。同一 GWM 在其中多个任务上达到或超越特定域基线方法，并展示出对新未见任务的强零样本/少样本能力。这体现了 GWM 作为通用世界模型在多样化数据结构上的潜力。

### 代码实现要点

- **通用消息传递**：Transformer-style attention + 动态边构建，支持结构化与非结构化节点混合
- **动作节点**：任务定义为动作节点，与相关节点通过直接引用或 embedding 相似度连接
- **GWM-T / GWM-E**：两种多模态融合策略——text token 统一 vs embedding 统一
- **代码开源**：GitHub 提供 GWM 实现

### 关键创新点

- **图世界模型新范式**：首次将世界模型扩展到图结构数据
- **动作节点统一任务**：以图的节点-边机制统一表达多样化任务
- **多模态图建模**：通过 token 化或嵌入统一处理任意模态输入
- **跨域零样本泛化**：在未见任务上展示强迁移能力
