# Object-Centric Modeling — 对象中心建模

::: tip 范式定义
Object-Centric Modeling 是 State Transition 范式中一条独特且富有哲学意味的子路线。它将世界建模为**一组独立对象**之间的交互动力学，而非一个单一的整体状态向量。核心主张是：**世界的组合结构是先验知识**——场景由对象组成，对象具有独立属性，对象间通过物理规则交互——好的世界模型应该显式地将这种结构编码到架构中。
:::

## 设计哲学：为什么需要对象？

在传统的隐状态空间模型中（如 Dreamer 系列），世界状态是一个无结构的向量 $s_t \in \mathbb{R}^d$。这种表示虽然紧凑高效，但面临两个结构性缺陷：

1. **组合泛化困难**：模型在「两个方块」的场景中训练，难以泛化到「三个方块」——因为状态向量没有内置的组合性和对象概念。

2. **可解释性缺失**：无法从状态向量中提取「有几个物体，各自在哪里，相互之间如何影响」的结构化信息，导致模型的推理过程不可审计。

Object-centric 建模通过将世界分解为**对象**及其**关系**来解决这两个问题。在这个框架中，世界的动力学由两部分组成：
- **对象内动力学**：单个对象的属性如何随时间变化（如位置、速度、外观）
- **对象间动力学**：对象如何通过交互影响彼此（如碰撞、支撑、遮挡）

## 核心技术：Slot-based 表示

Object-centric VWM 依赖 **Slot-based 表示**来编码对象信息。Slot 是一组可学习的「对象槽」，每个槽负责表示一个对象（或一个背景区域）。Slot 表示通过竞争性注意力机制从视觉观测中提取：

- **Slot Attention**（原始机制）：通过迭代竞争性注意力，将视觉特征「分配」给不同的 slot，每个 slot 自然地收敛到表示一个独立对象
- **SlotSSM**：将 slot 机制与状态空间模型（SSM）结合，在保持对象分离的同时实现高效的时序建模

Slot-based 表示的优势：
- 天然支持**变数量对象**（通过增加/减少 slot 数量）
- 自动实现**对象级别的注意力**（不同对象间的交互通过 slot 间的 attention 建模）
- 便于引入**对象级别的物理约束**（如质量、速度的守恒）

## 核心代表工作

### SlotFormer（2023-2024）

SlotFormer 是该路线的**奠基性工作**。它首次将 slot-based 对象表示与 Transformer 时序模型结合，构建了一个完整的对象中心世界模型。

SlotFormer 的架构：
- **Slot Extractor**：使用 Slot Attention 从每帧图像中提取对象 slot
- **Slot Transformer**：因果 transformer 处理 slot 序列，预测未来的 slot 状态
- **Slot Decoder**：将预测的 slot 解码回图像空间（通过 Spatial Broadcast Decoder）

SlotFormer 的核心发现是：**对象级别的预测比帧级别的预测更稳定、更可泛化**。模型在「两个球碰撞」的场景中训练后，可以合理地预测「三个球碰撞」的场景——这是因为 slot transformer 学习了「对象交互的通用规则」而非「特定数量对象场景的视觉模式」。

SlotFormer 还在无监督对象发现方面做出了重要贡献——模型可以在完全没有对象标注的视频中，自动学习将场景中的物体分离为独立的 slot。这种涌现的对象分离能力是该路线最吸引人的特性之一。

### slotSSM（2024-2025）

slotSSM 将 Slot-based 表示与**状态空间模型（SSM）**结合，旨在解决 SlotFormer 的一个关键瓶颈：Transformer 的 O(n²) 复杂度在处理长序列时计算负担过重。

slotSSM 的架构：
- **Slot Attention** 用于对象提取
- **Slot-wise SSM** 用于每个对象的独立动力学建模
- **Slot-Interaction SSM** 用于对象间的交互建模

SSM（状态空间模型，如 Mamba/S4）的关键优势是其 O(n) 复杂度，这使 slotSSM 能够处理比 SlotFormer长得多的交互序列。在需要长期对象追踪的任务中（如「物体放了 10 分钟后，它还在那里吗？」），slotSSM 展示出明显的优势。

### SSWM（Slot Structured World Models, 2024）

SSWM 将对象中心建模的**实用价值**推到了新的高度。它将 slot-based 世界模型与强化学习直接结合，证明了对象中心表示可以显著提升 RL 的样本效率：

- **对象感知的奖励函数**：奖励可以在对象层面定义（如「将红色方块放到蓝色方块上」），而非基于像素模式
- **对象感知的规划**：规划器可以在对象层面进行推理（哪些对象需要移动？如何移动？）
- **OOD 泛化**：在训练中未见过的对象配置下，模型展示出比无结构世界模型更强的泛化能力

SSWM 的一个重要发现是：**对象中心世界模型的样本效率优势随场景复杂度递增**。在简单场景（<3 个对象）中，对象中心表示相比 Dreamer 的增量不大；但在复杂场景（>10 个对象）中，对象中心的优势变得显著——传统隐状态模型需要指数级更多的样本才能学到对象间的交互。

### Object-Centric World Model for Language-Guided Manipulation（2025）

该工作将对象中心建模与**自然语言指令**结合，实现了语言引导的对象级操作：

- **Language-Conditioned Slot Selection**：语言指令「拿起红色的杯子」被解析为对特定 slot 的选择和操作
- **Slot-Level Affordance**：每个 slot 不仅包含对象的视觉属性，还包含其可操作性（能被拿起、能被推动、是否易碎等）
- **Compositional Task Generalization**：模型可以组合已知的对象操作完成新的任务（「将杯子放在书旁边」= 对「杯子」slot 的移动 + 参照「书」slot 的位置）

## 对象中心建模 vs 隐状态建模的核心对比

| 维度 | Object-Centric | Latent State-Space |
|------|---------------|-------------------|
| **状态表示** | 对象 slot 集合 | 单一状态向量 |
| **组合泛化** | **强**（对象数量可扩展） | 弱（固定维度向量） |
| **可解释性** | **高**（每个 slot 可解释为一个对象） | 低（向量不可解释） |
| **计算复杂度** | O(K²)，K=对象数量 | O(1)（对于固定维度向量） |
| **对象发现** | 需要学习（无监督 slot attention） | 无需（无显式对象） |
| **场景复杂度处理** | 适用中到高复杂度 | 适用低到中复杂度 |
| **训练数据量需求** | 中-高（需要学习对象分离） | 低-中 |

## 组合泛化：对象中心的皇冠珠宝

组合泛化（Compositional Generalization）是对象中心建模最核心优势。它的含义是：训练在一个场景配置下的模型，能够泛化到训练中未见过的配置。

这在隐状态模型中几乎不可能——因为模型学到了的是一个整体的「场景→下一状态」的映射，当场景中对象的数量或排列方式改变时，该映射不再成立。而对象中心模型学到的是「对象的通用行为规则」（如物体在重力下会下落，碰撞会导致速度交换），这些规则**与场景中对象的数量和排列无关**。

SlotFormer 的实验提供了一个令人信服的例子：模型在「2 个球碰撞」的视频上训练，展示了在「3 个球碰撞」和「4 个球碰撞」场景上的合理预测能力——尽管从未见过这些配置。这是「真正的物理理解」和「视觉模式记忆」之间的关键区别。

## 对象中心建模的局限

1. **对象发现的不可靠性**：Slot Attention 并非总是能正确地分离对象。当对象外观相似（如两个白色球）、或被严重遮挡时，slot 分配容易出错。错误的 slot 分配会向下游传播，导致灾难性的预测失败。

2. **背景和处理**：Slot-based 方法通常需要为「背景」分配专门的 slot，但背景并非一个明确定义的「对象」。如何处理场景中不归属于任何对象的像素是一个持久的技术难题。

3. **对象交互的复杂性爆炸**：当对象数量 K 增加时，对象间可能的交互组合以 O(K²) 增长。即使模型在原理上可以泛化，其预测质量可能随 K 上升而下降。

4. **与现实世界视觉的鸿沟**：当前的 slot-based 方法在合成场景（简单背景、鲜明对象）中工作良好，但在真实世界图像中（纹理丰富、光照复杂、对象边界模糊）的鲁棒性仍待验证。

## 总结与展望

Object-Centric Modeling 代表了世界模型中最具雄心的设计理念之一——它试图将人类对物理世界的结构化理解（「世界由对象组成」）直接编码到模型架构中。SlotFormer、slotSSM 和 SSWM 等工作在合成场景中证明了这一理念的价值，尤其是其令人瞩目的组合泛化能力。

但要实现从合成场景到真实世界的跨越，该路线仍需克服对象发现不可靠这一根本瓶颈。一个可能的方向是：借助大规模预训练的基础视觉模型（如 SAM、DINOv2）来提供「对象分割」作为先验，而非让世界模型从零开始学习对象分离。
